# Actions

Actions types and their purpose

NOTE - middleware here refers to dispatchUserMiddleware in Store

Properties:
Name Type Description
UPDATE_FIREBASE_USER

TODO;
GET_MONGO_USER

payload: {id:String} - used to fetch mongo user by id - passes through middleware
CREATE_USER

payload: {body:userObject} - calls User.createUser - passes through middleware
LOGIN_USER

payload: {type:String,body:{email,password}} - calls User.signInUser - fetches firebase and mongo user objects - passes through middleware
CREATE_NOTIFICATION

payload: Notification - dispatched by createNotification in Message.js - avoid dispatching this action directly and only use functions in Message.js
DISMISS_NOTIFICATION

payload:id - dispatched by dismissNotificationById in Message.js- avoid dispatching this action directly and only use functions in Message.js
SET_LOADING

payload : Boolean- sets state.loading
SET_INITIAL_LOAD

payload : Boolean- sets state.initialLoading
LOGOUT_USER

Logs out firebase user and sets all user objects and any related user data (such as uploaded singles) to null
WAIT_SINGLES_SUBMIT

payload : Boolean- sets state.waitSinglesSubmit
WAIT_ALBUM_SUBMIT

payload : Boolean- sets state.waitSinglesSubmit //TODO - Seperate this into waitAlbumSubmit
SUBMIT_SINGLES

payload : body:formData - calls Singles.uploadSingle - passes through middleware
SUBMIT_ALBUM

payload : body:formData - calls Albums.uploadAlbum - passes through middleware
FORM_LOADING

payload : Boolean- sets state.formLoading
SEND_INVITE_EMAIL

payload : email - calls User.sendInviteEmail - passes through middleware
SET_USER_SINGLES

payload : Array<Objects> - stores fetched Singles uploaded by the user in state.singlesArray
SET_USER_ALBUMS

payload : Array<Objects> - stores fetched Albums uploaded by the user in state.albumsArray
