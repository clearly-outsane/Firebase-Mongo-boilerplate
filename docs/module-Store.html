# initialState

The initial state.

Loading indicators are used so our app waits till the async requests are finished. This enables us to not only show the user loading indicators - but it also prevents us from prematurely executing any logic that relies on something to be loaded first.

Properties:
Name Type Description
initialLoading boolean

true When the page first loads - firebase checks for any existing user session and accordingly fetches the mongo user. This is run only once when app is initially loaded. Further user related async operations are indicated by initialState.loading
loading boolean

false Indicates loading of firebase and/or mongo users

For example:

  • When a login or sign up form is submitted.
  • If a user object needs to be updated.
waitSinglesSubmit boolean

false Indicates loading when single/album forms are being uploaded
formLoading boolean

false Indicates loading on other miscellaneous forms such as password reset forms
